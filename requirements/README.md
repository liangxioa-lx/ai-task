## 总体规划（V0.1）

### 1. 产品目标

- 支持用户通过AI对话创建和调整任务（一次性任务、周期性任务）
- 支持任务全生命周期管理（创建、编辑、执行、追踪）
- 支持任务分层（简单任务/复杂任务），为后续流程编排预留能力
- 支持任务执行日志沉淀，便于复盘与问题排查
- 支持多模型来源与安全边界配置（模型、权限、可操作目录）

### 2. 四模块关系

- AI对话：负责自然语言交互、任务生成与调整建议
- 任务管理：负责任务可视化、操作入口与流程编排
- 执行记录：负责执行结果归档、排障信息沉淀、统计支持
- 设置：负责模型接入与系统级安全策略

### 3. 跨模块关键流程

1. 任务管理点击“创建任务” -> 填写基本信息 -> 跳转AI对话并携带草稿上下文 -> AI生成任务结构 -> 回写任务管理列表  
2. 任务管理点击“编辑任务信息”或“与AI沟通调整任务” -> 跳转AI对话并携带历史上下文 -> 保存后更新任务  
3. 任务管理点击“现在执行” -> 触发执行器 -> 生成执行记录 -> 回写任务执行次数与上次执行时间  
4. 设置更新模型能力、默认权限或目录白名单 -> 影响后续AI对话和任务执行行为
5. 复杂任务（后续版本）通过编排多个子任务（简单或复杂）执行，并基于子任务执行状态/执行结果进行流程判断

### 4. 优先级建议

- P0：AI对话建任务、任务管理主表、执行记录基础、设置基础（模型+默认权限+目录范围）、任务实体预留`executionStatus`与`executionResult`字段
- P1：任务调整建议质量提升、日志筛选统计、模型健康检查
- P2：复杂任务流程编排（组合简单/复杂任务）、高级权限策略、多环境配置、执行审计增强
